49 ## Dependencies
50 #### Simulators
51 * [**UWSim**](http://www.irs.uji.es/uwsim/) Undewater Simulator.
52  the last version is the kinetic one (even if tutorial stop at the groovy version, it is only needed to follow it substituting `groovy` with `kinetic`).
53 * [**FreeFloating Gazebo**](https://github.com/freefloating-gazebo) Plugin for gazebo and UWSim to introduce dynamics. It also contatin a dynamic controller (which I use) which can be use to "transform" position, velocity or force (it can be choosen the modality) to thruster and joints.
54  Both the source (freefloating_gazebo) and the demo (freefloating_gazebo) are needed. **NOTE** not used anymore, maybe useful for future works?
55 #### Code
56 * **ROS kinetic**, obviously.
57  It is used not only to communicate with simulator, so also if other simulator is used (no UWSim or Gazebo), it is needed.
58 * **boost** (already installed with ROS ??)
59  the famous c++ library
60 * [**eigen 3**](http://eigen.tuxfamily.org/index.php?title=Main_Page) c++ library to deal with matrices in a sort of matlab style.
61  It contains also a geometry part (Transformation/Rotation matrix and so on) but I don't use it because of "geometrical" computations are done with CMAT.
62 * **CMAT** (private? library of University of Genova)
63  Needed for core of TPIK (regularized pseudoinversions)
64 * **kdl** (should be already installed with ROS)
65  parse urdf model of robots to get Manipulator Jacobian and position of end effector from the state (ie, angles) of joints
66 * **tf** (already installed with default ros?) ros package to deal with transformations provided by the simulator (eg position of the vehicle respect world)
67 * **parse_urdf** (already installed ros)
68  to parse the urdf model (see kdl)
69 
70 #### Vision part
71 * **opencv 3.4** (`sudo apt install ros-kinetic-opencv`, AND/OR normal installation following website)
72  The core for all vision things
73 * **cv_bridge** ros package to convert openCV images into ros message images and viceversa
74 * **image_transport** (already included with default ros installation?)
75  to deal with ros topics which share images
76 * [**VISP 3.2**](https://visp.inria.fr/) Visual Servoing Platform
77  It is nicer to use than opencv when dealing with more robotic vision things like *model based stereo tracking of an object*
78 * [**PCL**](http://www.pointclouds.org/)

81 ## Solutions for some possible compiling issue:
82 #### eigen library not found
83 It may be cause by the fact that eigen is installed in a folder named eigen3 (and not eigen).
84 Solution:
85 ```bash
86 cd /usr/include #the folder where is eigen3
87 sudo ln -s /usr/include/eigen3/Eigen Eigen #create symbolic link
88 ```
89 #### cmat_defines.h problem
90 in cmat there is a defined constant `#define PI 3.14xxxx`
91 this is a problem because kdl library has a const int called PI.
92 Solution is to change (in cmat_defines.h) :
93 `#define PI 3.14xxxxx` in `const int PI 3.14xxxxx`
94 
95 #### VISP_HAVE_PCL not setted
96 there is a function .track() in the tracker.cpp. This function is declared in a .h which has VISP\_HAVE\_PCL flag before.
97 Even with pcl-ros installed, this flag can be unsetted.
98 Solution? Not sure if I have broken all (at the moment is works, but be sure pcl is installed (ros-pcl is ok)), but I simply added
99 ```c
100 #define VISP_HAVE_PCL
101 ```
102 in /opt/ros/kinetic/include/visp3/visp_modules.h
